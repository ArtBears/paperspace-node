(static) create(params, cb) → {objectt}

Source:
  • machines/create.js, line 6

Create a new Paperspace virtual machine. If you are using an individual account, you will be assigned as the owner of the machine. If you are a team administrator, you must specify who the machine should belong to, either by their user id, or their email address. If you pass a full set of user parameters, we will create a user within your team and assign them the machine. This action can only be performed by the account owner. (Team members cannot create machines themselves; only the team administrator may do so.)

Examples
paperspace.machines.create({
  location: 'nyc01',
  machineType: 'Air',
  size: 50,
  billingType: 'hourly',
  machineName: 'My Machine 1',
  templateId: 123, // optional - only if creating from a template
  templateModel: 'template', // optional - only if creating from a template
  teamId: 456, // optional - required if creating for an team member
  userId: 123, // optional - required if creating for an user
  email: 'example@example.com', // optional - if creating a new user
  password: 'secret123', // optional - if creating a new user
  firstName: 'Jon', // optional - if creating a new user
  lastName: 'Snow', // optional - if creating a new user
  adUsername: 'jonsnow1', // optional - if creating a new user for an Active Directory team
  notificationEmail: 'example@example.com', // optional - to send a notification here when ready
}, function(err, resp) {
  // handle error or http response
});
$ paperspace machines create \
    --location "nyc01" \
    --machineType "Air" \
    --size 50 \
    --billingType "hourly" \
    --machineName "My Machine 1" \
    --templateId 123 \
    --templateModel "template" \
    --teamId 456 \
    --userId 123 \
    --email "example@example.com" \
    --password "secret123" \
    --firstName "Jon" \
    --lastName "Snow" \
    --adUsername "jonsnow1" \
    --notificationEmail "example@example.com"
# HTTP request:
POST /machines/createMachinePublic {"location": "nyc01", "machineType": "Air", "size": 50, "billingType": "monthly", "machineName": "Jon Snow 1", "templateId": 123, "templateModel": "template", "teamId": 456, "userId": 123, "email": "example@example.com", "password": "secret123", "firstName": "Jon", "lastName": "Snow", "adUsername": "jonsnow1", "notificationEmail": "example@example.com"}
# Returns 201 on success

(static) destroy(params, cb)

Source:
  • machines/destroy.js, line 6

Destroy the machine with the given id. When this action is performed, the machine is immediately shut down and marked for deletion from the datacenter. Any snapshots that were derived from the machine are also deleted. Access to the machine is terminated immediately and billing for the machine is prorated to the hour. This action can only be performed by the user who owns the machine, or in the case of a team, the team administrator.

Examples
paperspace.machines.destroy({
  publicMachineId: pslg3mr3,
}, function(err, resp) {
  // handle error or http response
});
$ paperspace machines destroy --publicMachineId 123
# HTTP request:
POST /machines/pslg3mr3/deactivateMachine
# Returns 200 on success
